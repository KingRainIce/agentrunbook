# Agent Project Research Snapshot

Snapshot date: 2026-07-01 UTC.

| Project | Approx stars | What people like | What AgentRunbook adopts | What AgentRunbook avoids |
| --- | ---: | --- | --- | --- |
| AutoGPT | 185k | Goal-driven automation, approachable agent idea | Clear CLI, workflow automation, public examples | Unbounded autonomy and heavyweight hosting as the first experience |
| LangChain | 141k | Ecosystem, integrations, standard interfaces | Provider abstraction and composability | Huge dependency surface for simple workflows |
| browser-use | 102k | A practical agent capability with immediate value | Narrow, concrete use cases and simple quickstart | Broad claims without a crisp job-to-be-done |
| MetaGPT | 69k | Role-based multi-agent collaboration | Agent roles, goals, and SOP-style runbooks | Over-specific "AI company" metaphor as the only model |
| AutoGen | 59k | Multi-agent conversations, human collaboration, tooling | Specialized agents and handoff-friendly structure | Large layered architecture for small automations |
| CrewAI | 55k | Intuitive crews and role-based tasks | Declarative roles and task ordering | Hidden execution and complex setup for first-time users |
| smolagents | 28k | Lightweight code-first core | Small dependency-free runtime | Arbitrary code execution as the default mental model |
| OpenAI Agents SDK | 28k | Tools, guardrails, sessions, tracing | Guarded tools and traces | Vendor lock-in in the runbook format |

Positioning: AgentRunbook is not trying to beat large frameworks at breadth. It is a small, auditable layer for turning repeatable knowledge-work workflows into versioned agent runbooks.
